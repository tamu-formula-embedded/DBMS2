# J1772

**Source:** `Core/Src/dbms/j1772.c`, `j1772.h`

## Overview

The J1772 module reads the SAE J1772 EV charging inlet to determine whether a charging cable is physically connected and, if so, what current the EVSE is willing to supply. It controls the charge-enable relay that signals the EVSE to begin delivering power.

---

## Proximity Pilot (PP) — Cable Detection

The Proximity Pilot line uses a resistor network to indicate connection status. The DBMS reads PP as an analog voltage via the STM32 ADC, converting to millivolts. A voltage below 1600 mV indicates a cable is connected (`pp_connect = true`). This replaces an earlier digital GPIO approach and is more robust against noise.

---

## Control Pilot (CP) — Current Capacity

The EVSE drives the CP line with a 1 kHz PWM signal whose duty cycle encodes the maximum current it can supply. The duty cycle is read by a PWM capture interrupt (not in this file) and stored in `j1772.cp_duty_cycle`. `J1772ReadState()` converts this to amps using the J1772 formula: `maxCurrentSupply = duty_cycle × 0.6 A`.

---

## Charge Enable Sequencing

There is a deliberate 10-second delay before the charge-enable GPIO (PC10) is asserted after the CP PWM signal is first detected. This gives the EVSE time to complete its handshake and allows the firmware to confirm a stable connection before enabling the charger. The `pwm_ts` timestamp records when the CP PWM was first received, and `J1772SetChargeEnable()` is only called with `true` once 10 seconds have elapsed with both `pp_connect` and valid `pwm_recieved` still true.

If CP PWM is not received for 500 ms (`PWM_READ_TIMEOUT`), or if the PP voltage rises above the threshold, charge enable is immediately pulled low.

---

## Interactions

`J1772ReadState()` is called at the top of every `ChargingUpdate()` invocation. The [Charging](charging.md) module reads `j1772.pp_connect` to determine if a cable is connected, and `j1772.maxCurrentSupply` to power-limit the Elcon charge request. The charge-enable GPIO drives the EVSE signal hardware directly.
